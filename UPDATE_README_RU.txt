ОБНОВЛЕНИЕ ПРОВЕРКИ АКТИВНОСТИ
================================

1. Сделайте резервную копию bot.db и settings.json.
2. Установите зависимости: python -m pip install -r requirements.txt
3. Запустите проверку: python -m py_compile main.py db.py inactivity.py admin_activity_panel.py
4. Запустите бота обычным способом: python main.py
5. Напишите боту в личные сообщения: /activity
6. Выберите чат и настройте параметры.

ВАЖНО
- bot.db удалять не нужно: схема обновится автоматически при запуске.
- Для учёта всех сообщений отключите Privacy Mode у бота через BotFather.
- Бот должен быть администратором групп, чтобы надёжно проверять статус участников.
- Ушедшие/кикнутые участники удаляются из рабочей базы при событии выхода и при очередной проверке.

НАСТРОЙКИ
- Порог неактивности: сколько дней участник не писал.
- Интервал проверки: как часто проверяется выбранный чат.
- Повтор уведомления: через сколько часов повторять алерт о том же нарушении.
- Минимум сообщений: уведомлять, если после порогового периода участник написал меньше N сообщений. 0 отключает настройку.

ОБНОВЛЕНИЕ АДМИН-ПАНЕЛИ
------------------------
- Команды /admin и /activity открывают одну общую админ-панель.
- Старые кнопки «Неактивные» и «Неактивность» заменены кнопкой «📊 Активность чатов».
- Настройки выбранного чата защищены: менять их может владелец бота или глобальный администратор, который также является администратором выбранной Telegram-группы.
- Глобальный запрет владельца на изменение настроек распространяется и на настройки активности.
- Подмена chat_id в callback-кнопках блокируется проверкой выбранного чата в текущей FSM-сессии.
- Кнопка «🧪 Тест оповещения» отправляет тест только нажавшему администратору и не меняет историю реальных уведомлений.
- Кнопка «⚡ Реальная проверка» запускает настоящую проверку и может отправить реальные алерты всем выбранным получателям.

ОБНОВЛЕНИЕ БАЗЫ УЧАСТНИКОВ
---------------------------
- Перед каждой реальной проверкой неактивов бот сверяет всех известных участников
  выбранного чата через Telegram getChatMember.
- Статусы left/kicked/banned удаляются из bot.db, истории алертов и
  members_panel.sqlite3.
- В настройках чата появилась кнопка «🔄 Обновить базу участников».
  Она запускает ручную сверку и показывает отчёт.
- События выхода, кика и бана также обрабатываются сразу через chat_member update.
- Для получения этих событий бот должен быть администратором чата.
